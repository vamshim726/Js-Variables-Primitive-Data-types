2. Difference between undefined and null

In JavaScript, use null to explicitly indicate that a variable has no value or is intentionally empty. Use undefined when something is naturally missing, such as an uninitialized variable or a non-existing object property. null represents intentional absence, while undefined signals an expected missing value.

4. Research: defer vs async & document.write()

ðŸ”¹ defer vs async

  • defer: Script is downloaded in parallel but executed after HTML parsing is complete (in order).
  • async: Script is downloaded and executed as soon as it's ready, not in order.

ðŸ”¹ document.write() Method

document.write() writes HTML or text directly to the document during loading. It's discouraged after the page has loaded, as it can overwrite the whole DOM. Better alternatives include innerHTML or DOM manipulation.
